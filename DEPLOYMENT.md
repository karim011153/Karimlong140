# دليل الرفع على الاستضافة - Lingo App

## المتطلبات
- Node.js 18+ و npm
- مفتاح API من Google Gemini (اختياري للاستخدام المباشر من المتصفح)

## الإصلاحات المُطبقة

### 1. إصلاح مشاكل الأيقونات والصور
✅ تم تحويل جميع المسارات من مطلقة (`/icons/...`) إلى مسارات نسبية (`./icons/...`)
✅ تم تحديث `index.html`, `manifest.json`, `sw.js`
✅ تم نسخ جميع الأيقونات إلى مجلد `dist/icons/`

### 2. إصلاح مشاكل الاتصال بالـ API
✅ إزالة "القائمة البيضاء" المحددة لأسماء النطاقات
✅ التطبيق يحاول الاتصال بـ `/api/gemini` و `/api/tts` أولاً
✅ إذا فشل الاتصال بالخادم، يحاول الاتصال المباشر بـ Gemini API
✅ يعمل على أي استضافة (ساكنة أو ديناميكية)

### 3. تحديثات تطبيق Flutter
✅ تحديث الموديل إلى `gemini-1.5-flash` (أكثر استقراراً)

## خيارات الرفع

### الخيار 1: استضافة ديناميكية (مع Node.js)
مثل: Heroku, Railway, Render, Google Cloud Run

```bash
# 1. بناء المشروع
npm run build

# 2. رفع المشروع
# الخادم سيقدم الملفات من مجلد dist/ ويوفر API endpoints
```

### الخيار 2: استضافة ساكنة (Static Hosting)
مثل: Netlify, Vercel, GitHub Pages, Firebase Hosting

```bash
# 1. بناء المشروع
npm run build

# 2. رفع محتويات مجلد dist/ فقط
# ستحتاج إلى إدخال مفتاح Gemini API في الإعدادات المتقدمة للتطبيق
```

### الخيار 3: استضافة مخصصة (VPS/Dedicated Server)

```bash
# 1. تثبيت Node.js و npm
# 2. نسخ المشروع
# 3. تثبيت التبعيات
npm install

# 4. بناء المشروع
npm run build

# 5. تشغيل الخادم
npm start
# أو استخدام PM2 للتشغيل المستمر
pm2 start server.ts --name lingo
```

## متغيرات البيئة

إنشاء ملف `.env` أو `.env.local`:

```
GEMINI_API_KEY=your_api_key_here
```

## اختبار التطبيق محلياً

```bash
# تشغيل خادم التطوير
npm run dev

# سيكون متاحاً على http://localhost:5173
```

## ملاحظات مهمة

1. **الأيقونات**: تأكد من أن مجلد `dist/icons/` يحتوي على جميع الأيقونات قبل الرفع
2. **Service Worker**: قد تحتاج إلى مسح الـ Cache من المتصفح إذا واجهت مشاكل في الأيقونات
3. **CORS**: إذا كنت تستخدم استضافة ساكنة، تأكد من تفعيل CORS للاتصال المباشر بـ Gemini API
4. **مفتاح API**: يمكنك إدخال مفتاح API من الإعدادات المتقدمة في التطبيق

## استكشاف الأخطاء

### الأيقونات لا تظهر
- تحقق من أن مجلد `dist/icons/` يحتوي على الأيقونات
- امسح الـ Cache من المتصفح (Ctrl+Shift+Delete)
- تحقق من أن المسارات نسبية وليست مطلقة

### الـ API لا يستجيب
- تحقق من أن الخادم يعمل على المنفذ 3000
- تحقق من أن متغير `GEMINI_API_KEY` مضبوط بشكل صحيح
- جرب إدخال مفتاح API من الإعدادات المتقدمة

### Service Worker يسبب مشاكل
- امسح جميع الـ Caches من DevTools
- أعد تحميل الصفحة (Ctrl+F5)
- تحقق من أن `sw.js` يستخدم المسارات النسبية
